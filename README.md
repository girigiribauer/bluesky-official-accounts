# 公式アカウント移行まとめ

- 移行具合を可視化すると移行の促進になるのは明らかなのに、目の前の表だけ見て満足して手伝ってくれない人が大半
    - フォームから投稿しても直接インセンティブがあるわけでもない
    - Notion に参加してチェックするのは敷居が高すぎる
    - 自分の興味のない分野まで見るのは億劫
    - 一部の人たちに負荷がかかりすぎる
- 分野ごとに自治してもらい、あなたが興味を持つ分野が整えられてないのは手伝ってくれないから、というのも含めてすべて分野単位で可視化する

## ローカルの開発環境

### 初期化

```bash
make init
```

### 開発環境の起動

```bash
make dev
```

### テスト

```bash
make test
```

### ビルド

```bash
make build
```

## ローカルのDB操作

起動は `make dev` が兼ねています。

### 停止

```bash
make stop
```

### 空にしつつスキーマ適用

```bash
make reset
```

## 本番DBへのマイグレーション適用

初回のみ本番プロジェクトと紐付ける（`<project-ref>` は Supabase ダッシュボードの URL から確認）。

```bash
npx supabase link --project-ref <project-ref>
```

マイグレーションを本番に適用する（デプロイより先に実行すること）。

```bash
npx supabase db push
```
