# モデレーションシステムへの段階的な移行計画

## 概要

旧カテゴリーシステムから新しい『分野別モデレーションシステム』への移行計画。
段階的なアプローチにより、リスクを最小化しながら、コミュニティ主導の『分野別モデレーションシステム』へ移行する。

## フェーズごとの計画

### フェーズ1: 投稿フォーム見直しとフロントエンド刷新

協力者を今後より多く求めていく必要があるため、
現在のものが有用であると再認識してもらう必要がある。

そこで、一旦 Notion での管理はそのままに、
そこに至るまでの投稿フォームの自前化（Notion API を叩きつつバリデーションする）を行う。
また、同時に企業・組織向けに新規ページを追加しつつフロントエンドを刷新して、同時に協力者を募る。

### フェーズ2: 計画の策定・要件定義

新しいシステムの設計方針を固め、実装に必要な要件を定義する。

- 分野の確定
- 旧カテゴリーからのマッピング作成
- モデレーションシステムのルール設計
- マイグレーション計画の策定
- モデリングの仮定義
  - 必要なエンティティ（ユーザー、投稿、評価、監査ログ等）の洗い出し
- UI/UX・画面要件の定義
  - 画面遷移図、必要な画面リスト、主要な操作フロー

### フェーズ3: モデレーションシステムの構築

ベータ版として動作するシステムを構築する。

- データベーススキーマの実装
- 認証・認可システムの実装
- 投稿・承認フローの実装
- モデレーター管理機能の実装
- ポイント制度の実装
- 異議申し立て機能の実装
- 通知システムの実装(サイト内通知のみ)

### フェーズ4: ベータ版運用

実際の運用を通じてシステムを検証・改善していく。

- 初期モデレーターの任命（管理者が分野ごとに1〜2名）
- ベータ版公開

### フェーズ5: 正式公開

全ユーザーに新システムを公開し、旧カテゴリーからのデータを完全移行する。
