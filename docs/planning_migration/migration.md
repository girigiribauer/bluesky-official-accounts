# モデレーションシステムへの段階的な移行計画

## 概要

旧カテゴリーシステムから新しい『分野別モデレーションシステム』への移行計画。
段階的なアプローチにより、リスクを最小化しながら、コミュニティ主導の『分野別モデレーションシステム』へ移行する。

## フェーズごとの計画

### フェーズ1: 投稿フォーム見直しとフロントエンド刷新（済）

協力者を今後より多く求めていく必要があるため、
現在のものが有用であると再認識してもらう必要がある。

そこで、一旦 Notion での管理はそのままに、
そこに至るまでの投稿フォームの自前化（Notion API を叩きつつバリデーションする）を行う。
また、同時に企業・組織向けに新規ページを追加しつつフロントエンドを刷新して、同時に協力者を募る。

### フェーズ2: モデレーションシステムの要件定義

新しいシステムの設計方針を固め、実装に必要な要件を定義する。

- 分野の確定
- モデレーションシステムの仕様の確定
- アーキテクチャ構成の設計
- モデリング
- UI/UX・画面設計

### フェーズ3: MVPの構築と運用開始

最小限の機能でモデレーションシステムを稼働させ、実運用を始める。

- データベーススキーマの実装（最小限）
- Blueskyアカウントによる認証の実装
- 投稿・承認フローの実装（レビュー・公開・却下）
- 操作ログの実装
- 限定的な協力者向けにMVP公開
- 利用規約の用意

### フェーズ4: ベータ版（ゲーミフィケーション導入）

ロール・ポイント・ゲーミフィケーション要素を導入し、コミュニティ主導の運用が成立するかを検証する。

- 段階制度（レベルデザイン）の実装
- ポイント制度の実装
- 異議申し立て機能の実装
- 通知システムの実装（サイト内通知のみ）
- 初期モデレーターの任命（管理者が分野ごとに1〜2名）
- ベータ版公開・検証

### フェーズ5: 正式公開

ベータ版での検証を経て、全ユーザーに新システムを公開し、旧カテゴリーからのデータを完全移行する。
